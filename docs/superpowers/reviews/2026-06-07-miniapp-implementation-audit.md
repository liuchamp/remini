## 4. 代码复用策略

| Web端资源 | 复用方式 | 备注 |
|-----------|----------|------|
| 后端 API | 100% 复用 | Header加 `X-Client-Type: miniapp` |
| Prisma Schema | 100% 复用 | 数据模型完全一致 |
| 业务逻辑 | 服务层参考 | 改写为Taro兼容(无Node API) |
| TypeScript 类型 | 90% 复用 | 从remxv1提取共享包 |
| 常量/枚举/工具函数 | 80% 复用 | format/validate等直接移植 |
| UI组件 | 0% 复用 | Web:Tailwind+Radix → Mini:NutUI+Taro |
| 路由/页面 | 0% 复用 | React Router 7 SSR → Taro页面栈 |
| 状态管理 | 模式复用 | Context/Reducer → Zustand |
| WebSocket | 协议复用 | ws:// → wx.connectSocket |

## 5. 预估工期与风险

### 工期预估

| 阶段 | 功能点 | 工时 | 累计 |
|------|--------|------|------|
| Phase 1 | KYC/钱包/物流 | 5-6 天 | 1 周 |
| Phase 2 | 社区/营销/通知 | 6-8 天 | 2-3 周 |
| Phase 3 | i18n/搜索/分享/性能 | 4-5 天 | 3-4 周 |
| **缓冲/联调/提审** | - | **1 周** | **4-5 周** |

### 关键风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 微信审核不通过(支付/KYC/分享) | 上线延迟1-2周 | 提前准备测试账号/隐私协议/类目资质 |
| Canvas海报真机性能差 | 生成失败/卡顿 | 降级为服务端生成图片URL |
| WebSocket小程序不稳定 | 消息延迟/丢失 | 已有轮询降级，调优间隔(30s/60s) |
| 主包超2MB | 无法上传 | 分包优化/动态import/CDN外置大资源 |
| i18n文案遗漏 | 线上显示key | 构建时自动扫描未翻译key报警 |
