## ADDED Requirements

### KYC 等级体系

#### 四级认证体系定义
WHEN 用户进入「身份认证」页面
THEN 系统展示 L0/L1/L2/L3 四级认证状态：L0（未实名，所有受限）、L1（手机验证，基础权限）、L2（身份证认证，交易权限）、L3（活体检测，营销权限）；每级以图标标识完成状态（待认证/认证中/已认证），并说明下一级可解锁的权益

#### 等级提升限制
WHEN 用户申请提升至更高级别
THEN 系统校验前一级是否已完成（L1→L2→L3 逐级递增），未完成低级别认证则不展示高级别认证入口；每级认证完成后实时刷新用户等级状态，并广播等级变更事件供业务组件响应

### 手机验证（L1）

#### 手机号绑定验证
WHEN 用户申请完成 L1 认证（默认使用当前登录手机号）
THEN 前端向用户展示已绑定手机号（中间 4 位脱敏），请求用户点击获取验证码；后端发送 6 位数字验证码至该手机号，用户输入验证码后调用 /api/kyc/verify-phone 接口完成验证

#### 设备指纹采集
WHEN 用户提交 L1 验证请求
THEN 前端携带当前设备指纹数据（设备 ID、系统版本、Taro UA、网络运营商信息）一并提交至后端；后端将设备指纹持久化关联至用户的 L1 认证记录，用于后续风控比对

### 身份证 OCR 认证（L2）

#### 身份证正反面拍照与 OCR 识别
WHEN 用户申请 L2 认证并授权相机权限
THEN 前端引导用户拍摄身份证正面（人像面）和反面（国徽面），调用 Taro.chooseMedia 选择/拍照；上传至后端 /api/kyc/ocr-id-card 接口，后端调用 OCR 服务识别身份证信息（姓名、身份证号、有效期、住址等），返回识别结果

#### 自动填充与人工确认
WHEN OCR 识别成功返回姓名、身份证号等字段
THEN 前端自动填充识别结果至表单，允许用户核对并修改（若识别有误）；用户确认无误后提交认证申请至 /api/kyc/submit-id-card 接口；若识别置信度低于 90%，标记为「需人工审核」

#### 人工审核流程
WHEN 用户提交身份证认证申请（含 OCR 结果和原始图片）
THEN 后端记录申请进入审核队列，状态变更为「审核中」；审核人员在后端管理后台查看 OCR 结果与原始图片对比，操作通过/驳回；审核结果通过订阅消息/短信通知用户；若驳回，用户需重新拍摄并提交

### 活体检测（L3）

#### 眨眼/摇头/张嘴检测
WHEN 用户申请 L3 认证并已通过 L1 + L2
THEN 前端引导用户进入活体检测页面，展示检测动作提示（随机 3 个动作：眨眼、左右摇头、张嘴）；用户按照提示完成动作，前端通过摄像头实时采集视频流并上传至后端 /api/kyc/liveness-detection 接口；后端调用活体检测算法验证视频真实性

#### 摄像头权限申请
WHEN 用户首次进入活体检测页面
THEN 前端调用 Taro.authorize 申请摄像头权限（scope.camera）；若用户拒绝，展示引导弹窗说明「活体检测需要使用摄像头，请前往设置开启」，并提供跳转系统设置入口；用户再次拒绝则无法继续 L3 认证

#### 视频录制上传
WHEN 活体检测动作采集完成
THEN 前端将录制的短视频片段（通常 3-5 秒，每动作一段）上传至后端；后端校验视频质量（时长 > 1 秒、分辨率 > 480p、非静态图片注入），通过活体检测算法返回结果；检测通过则升级用户至 L3，失败则提示用户重新录制

### KYC 守卫

#### 发布商品需 L1 以上
WHEN 未认证（L0）用户点击「发布商品」
THEN 前端校验当前用户 KYC 等级 < L1，弹窗拦截并引导用户完成 L1 手机验证；用户在弹窗内可直接跳转至 KYC 认证页面，完成 L1 认证后返回发布页继续操作

#### 提现需 L2 以上
WHEN 用户发起提现请求
THEN 后端校验用户 KYC 等级 < L2，返回 403 及提示「提现需要完成 L2 实名认证」；前端收到响应后展示认证引导页，告知用户需补传身份证信息方可提现

#### 营销功能需 L3
WHEN 用户参与营销活动（领优惠券、积分兑换、邀请奖励兑换）
THEN 后端在活动参与接口校验用户 KYC 等级 < L3，返回业务错误「此活动需完成 L3 身份认证」；前端展示 L3 活体检测引导入口，用户完成认证后自动恢复参与资格

#### 合规守卫校验时机
WHEN 用户执行任何受 KYC 等级限制的操作
THEN 后端在相应 API 中间件层进行 KYC 等级校验，确保限制在服务端不依赖前端判断；前端同时在 UI 层面隐藏/禁用未达等级的功能按钮，避免用户误触；前后端双重校验确保合规
